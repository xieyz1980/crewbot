## 🔍 10分钟检查流程

```
每10分钟执行：
  ├─ 1. Git状态检查
  │   ├─ books仓库：最新提交、文件变更
  │   └─ crewbot仓库：最新提交、代码行数
  │
  ├─ 2. 产出验证
  │   ├─ 字数统计（wc -l/wc -w）
  │   ├─ 文件存在性检查（ls -la）
  │   └─ 质量抽检（读取文件头/尾）
  │
  ├─ 3. 进度评估
  │   ├─ 对比计划时间表
  │   ├─ 判断是否滞后
  │   └─ 判断是否提前完成
  │
  ├─ 4. 行动决策
  │   ├─ 正常推进 → 记录日志
  │   ├─ 发现滞后 → 飞书提醒
  │   └─ 提前完成 → 启动下一步
  │
  └─ 5. 记录监督日志
```

## ⚠️ 摸鱼判定标准

### 人类组员

| 行为 | 判定 | 处理措施 |
|------|------|----------|
| 1小时无Git提交 | 🟡 疑似摸鱼 | 飞书提醒询问进度 |
| 2小时无产出 | 🔴 确认摸鱼 | 老板汇报+催促 |
| 产出质量下降 | 🟡 需要关注 | 提供协助建议 |
| 提前完成任务 | 🟢 优秀 | 立即分配下一步 |

### AI Agent

| 行为 | 判定 | 处理措施 |
|------|------|----------|
| 任务队列堆积 | 🔴 执行阻塞 | 立即排查原因 |
| 10分钟无产出 | 🟡 效率低下 | 重启Agent或调整参数 |
| 产出质量不达标 | 🔴 质量问题 | 回滚重写 |
| 提前完成 | 🟢 优秀 | 立即分配新任务 |

## 🚀 提前完成处理机制

**判定标准**：
- 任务完成时间 < 计划时间 × 80%

**自动处理**：
1. 立即验证产出质量
2. 质量OK → 自动分配下一步任务
3. 更新MEMORY_INDEX.md
4. 飞书通知老板任务提前完成

**示例**：
```
07:00 检查：第5章框架生成任务

07:10 处理：自动启动"第5章初稿生成"任务
07:10 通知：飞书汇报老板"任务提前完成，已启动下一步"
```
