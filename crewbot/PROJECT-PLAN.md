## 🏗️ 技术架构

### 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    CrewBot Core                         │
│                 (编排调度引擎)                            │
└────────────────────┬────────────────────────────────────┘
                     │
    ┌────────────────┼────────────────┐
    │                │                │
┌───▼────┐    ┌─────▼──────┐   ┌─────▼──────┐
│ Agent  │    │   Agent    │   │   Agent    │
│ 写作   │    │   编程     │   │   分析     │
└───┬────┘    └─────┬──────┘   └─────┬──────┘
    │               │                │
    └───────────────┼────────────────┘
                    │
         ┌──────────▼──────────┐
         │    One API Router   │
         │   (智能模型选择)     │
         └──────────┬──────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐    ┌─────▼─────┐   ┌────▼────┐
│ GPT-4 │    │ Claude-3  │   │ 本地模型 │
│ 系列  │    │   系列    │   │ llama3  │
└───────┘    └───────────┘   └─────────┘
```

### 核心模块

**1. 编排引擎 (Orchestrator)**
- 负责任务分解和Agent调度
- 支持DAG（有向无环图）和状态机两种模式
- 动态负载均衡

**2. Agent框架**
- 标准化Agent接口
- 支持Python/TypeScript编写Agent
- 内置常用Agent（写作、编程、分析等）

**3. One API Router**
- 智能模型选择算法
- 成本优化策略
- 故障转移机制

**4. 通信总线**
- Agent间消息传递
- 状态同步
- 事件驱动架构

**5. Web UI**
- 可视化配置
- 实时监控
- 日志查看

### One API Router算法

```python
class ModelRouter:
    def select_model(self, task: Task) -> Model:
        """智能选择模型"""
        
        # 1. 任务分类
        task_type = self.classify_task(task)
        
        # 2. 成本预算检查
        budget = task.budget
        
        # 3. 模型选择策略
        if task_type == TaskType.CODE:
            # 代码任务：优先GPT-4，Claude备用
            return self.select_by_cost(
                ["gpt-4", "claude-3-opus"], 
                budget
            )
        elif task_type == TaskType.WRITING:
            # 写作任务：Claude擅长长文本
            return self.select_by_quality(
                ["claude-3-opus", "gpt-4"],
                budget
            )
        elif task_type == TaskType.SIMPLE:
            # 简单任务：使用轻量模型节省成本
            return "gpt-3.5-turbo"
        
        # 4. 故障转移
        if not self.check_availability(selected_model):
            return self.fallback_model(selected_model)
```
